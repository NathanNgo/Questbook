# Questbook

### Front End Environment Setup

In addition to the setup steps detailed in the front end's `README`, an additional step is required to properly initialise the development environment.

1. Navigate to the `frontend` directory
2. Copy `.env.template` and rename new version to `.env`
3. Enter relevant environment variables into the file {i.e: `VITE_BACKEND_URL`...etc }
4. Save the file and restart the front end server. Everything should be working as normal

### OpenAPI Schema Documentation and Schema Generation

The backend uses Huma to generate API docs (accessible via the `/docs` api endpoint \["BACKEND_URL/docs"\]).

These docs can also be used to generate the schemas used by the front end. These are pulled from the docs endpoint to create the `src/types/api.ts` file. This file can be regenerated by running `pnpm run generate:api` from the frontend directory.
